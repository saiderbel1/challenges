# Procurement Request System - Development Guide

## Overview

The Procurement Request System is a Streamlit-based web application that automates the processing of procurement requests. Users can upload PDF documents containing procurement information, which are then parsed using an LLM (GPT-4) to extract structured data. The system supports two user roles: regular users who submit requests, and managers who review and update request statuses.

## Architecture

```
procurement_system/
├── streamlit_app.py          # Main entry point, routing, authentication
├── app_cli.py                # CLI version (alternative interface)
├── main.py                   # Original script (legacy)
│
├── services/                 # Application orchestration layer
│   ├── __init__.py
│   └── procurement_app.py    # ProcurementApp - central service class
│
├── intake_management/        # LLM extraction and data models
```

## Key Components

### 1. ProcurementApp (`services/procurement_app.py`)

Central orchestration class that provides a unified interface to all services.

```python
class ProcurementApp:
    # PDF Processing
    def extract_text_from_pdf_bytes(pdf_bytes: bytes) -> str
    def extract_procurement_data(pdf_text: str) -> ExtractedProcurementData

    # Request Management
    def save_request(request: ProcurementRequest) -> int
    def get_user_requests(requestor_name: str) -> list[tuple[int, ProcurementRequest]]
    def get_all_requests() -> list[tuple[int, ProcurementRequest]]
    def update_request_status(request_id: int, status: RequestStatus) -> bool

    # Utilities
    def get_commodity_groups() -> list[tuple[int, str]]
    def get_commodity_group_name(group_id: int) -> str
```

### 2. IntakeManager (`intake_management/intake_manager.py`)

Handles LLM-based extraction of procurement data from PDF text.

- Uses LangChain with OpenAI GPT-4
- Structured output via Pydantic models
- Contains `COMMODITY_GROUPS` constant (50 categories)
- Prompt engineering for accurate extraction

### 3. Data Models (`intake_management/types.py`)

```python
class RequestStatus(str, Enum):
    OPEN = "Open"
    IN_PROGRESS = "In Progress"
    CLOSED = "Closed"

class OrderLine(BaseModel):
    position_description: str
    unit: str
    unit_price: float

class ExtractedProcurementData(BaseModel):
    # Fields extracted by LLM (excludes user input)

    vendor_name: str
    vat_id: str
    commodity_group: int
    order_lines: list[OrderLine]

class ProcurementRequest(BaseModel):

    title: str
    vendor_name: str
    vat_id: str
    commodity_group: int
    order_lines: list[OrderLine]
    total_cost: float
    status: RequestStatus = RequestStatus.OPEN
```

## Database Schema

```sql
-- requests table
CREATE TABLE requests (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    requestor_name TEXT NOT NULL,
    title TEXT NOT NULL,
    vendor_name TEXT NOT NULL,
    vat_id TEXT NOT NULL,
    requestor_department TEXT NOT NULL,
    commodity_group INTEGER NOT NULL,
    total_cost REAL NOT NULL,
    status TEXT NOT NULL DEFAULT 'Open',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- order_lines table
CREATE TABLE order_lines (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    request_id INTEGER NOT NULL,
    position_description TEXT NOT NULL,
    unit TEXT NOT NULL,
    unit_price REAL NOT NULL,
    amount REAL NOT NULL,
    total_price REAL NOT NULL,
    FOREIGN KEY (request_id) REFERENCES requests(id) ON DELETE CASCADE
);
```

## Extending the Application

### Adding a New Status

1. Update `RequestStatus` enum in `intake_management/types.py`
2. Update status color mapping in views (`_get_status_color` function)

### Adding New Fields to Requests

1. Add field to `ProcurementRequest` in `types.py`
2. If LLM-extracted: also add to `ExtractedProcurementData` and update prompt
3. Update database schema in `database_manager.py`
4. Update repository methods in `request_repository.py`
5. Update views to display/edit the new field

### Adding a New View

1. Create new file in `views/` directory
2. Implement `render(app: ProcurementApp)` function
3. Export in `views/__init__.py`
4. Add navigation in `streamlit_app.py`

### Modifying the LLM Prompt

Edit `EXTRACTION_PROMPT` in `intake_management/intake_manager.py`:

```python
EXTRACTION_PROMPT = ChatPromptTemplate.from_messages([
    ("system", """Your system prompt here..."""),
    ("human", "Your human prompt here..."),
])
```

### Adding New Commodity Groups

Update `COMMODITY_GROUPS` dictionary in `intake_management/intake_manager.py`.
